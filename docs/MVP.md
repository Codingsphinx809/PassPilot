# PassPilot MVP

Version: 1.0

Status: Planning

---

# Purpose

This document defines exactly what will be included in PassPilot Version 1.0.

Anything not listed here is considered **out of scope** and will be evaluated after the MVP has been successfully released.

The objective of the MVP is to deliver a polished, reliable, and valuable certification learning platform—not a feature-complete one.

---

# Success Criteria

The MVP will be considered successful if a learner can:

1. Create an account.
2. Choose a certification.
3. Follow a structured study plan.
4. Complete lessons and quizzes.
5. Track their progress.
6. Take a mock exam.
7. Feel confident enough to schedule and pass their certification exam.

---

# Target Audience

The initial release targets professionals preparing for:

- ITIL Foundation V5

Future certifications will be added after validating the platform.

---

# Core Features

## Authentication

- User registration
- Email login
- Password reset
- Secure authentication using Supabase

---

## User Profile

- Name
- Avatar
- Certification progress
- Study statistics
- Account settings

---

## Onboarding

- Welcome experience
- Certification selection
- Experience level
- Target exam date
- Weekly study availability

Generate an initial personalized study plan.

---

## Dashboard

Display:

- Current course
- Today's study recommendation
- Progress
- Confidence Score
- Study streak
- Upcoming milestones

---

## Course Engine

Support:

- Certification
- Modules
- Lessons
- Learning objectives

The architecture must support multiple certifications without significant changes.

---

## Lesson Player

Lessons should include:

- Rich text
- Images
- Diagrams
- Knowledge checks
- Completion tracking

---

## Quiz Engine

Support:

- Single choice
- Multiple choice
- Immediate feedback
- Detailed explanations
- Review mode

---

## Progress Tracking

Track:

- Lesson completion
- Quiz performance
- Module completion
- Overall certification progress

---

## Confidence Score

Provide learners with an estimate of their readiness based on:

- Quiz accuracy
- Topic mastery
- Study consistency
- Mock exam performance
- Knowledge retention

The Confidence Score should encourage improvement rather than discourage learners.

---

## Mock Exams

Include:

- Timed exams
- Randomized questions
- Detailed results
- Performance by topic
- Suggested review areas

---

## Settings

Allow users to manage:

- Profile
- Notifications
- Theme
- Account
- Privacy preferences

---

# Nice-to-Have Features

These may be included if time permits but are not required for the MVP:

- Flashcards
- Daily reminders
- Achievement badges
- Notes
- Bookmarks
- Dark mode
- Offline lesson caching

---

# Explicitly Out of Scope

The following features are intentionally excluded from Version 1.0:

- AI Tutor
- Voice interactions
- Enterprise portal
- Team management
- Instructor dashboard
- Community forums
- Leaderboards
- Marketplace
- Web application
- Apple Watch support
- Wearables
- Gamified challenges
- Multi-language support
- Advanced analytics
- Live tutoring

These features will be considered for future releases.

---

# Quality Standards

Before release, the MVP must be:

- Stable
- Responsive
- Accessible
- Secure
- Well documented
- Fully tested

No feature should be released in an unfinished state.

---

# Definition of Done

A feature is considered complete only when:

- Requirements are implemented.
- Code has been reviewed.
- Tests pass.
- Documentation is updated.
- Accessibility has been verified.
- No critical bugs remain.

---

# MVP Goal

The MVP is not intended to compete with every certification platform on day one.

Its purpose is to prove that PassPilot can help learners prepare for professional certifications through a focused, intuitive, and trustworthy learning experience.

Success will be measured by learner outcomes, user satisfaction, and the platform's ability to scale beyond ITIL into additional certifications.
