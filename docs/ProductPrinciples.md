# Product Principles

These principles define how every feature in PassPilot should be designed and built.

---

# 1. Learning Before Testing

PassPilot is not a question bank.

Questions reinforce learning—they do not replace it.

---

# 2. Confidence Over Completion

Finishing 100% of a course does not mean someone is ready.

Every feature should increase the learner's confidence through measurable progress.

---

# 3. Personalization First

No two learners are the same.

The platform should adapt study recommendations, review sessions, and practice based on each learner's strengths and weaknesses.

---

# 4. Mobile First

The mobile experience is the primary experience.

Every feature should feel natural on a phone before considering larger screens.

---

# 5. Simplicity Wins

Complexity should stay behind the scenes.

Users should never feel overwhelmed by the interface.

---

# 6. Explain Everything

Never tell a learner an answer is wrong without explaining why.

Every incorrect answer is a learning opportunity.

---

# 7. Respect the User's Time

Help users study smarter, not longer.

Every session should feel productive.

---

# 8. Measure Progress Meaningfully

Track:

- Knowledge
- Confidence
- Consistency
- Retention

instead of only completed lessons.

---

# 9. Build for Scale

Every feature should support multiple certifications without requiring major rewrites.

---

# 10. Delight the User

Professional software doesn't have to feel boring.

Small animations, thoughtful interactions, and polished design create a learning experience users enjoy returning to.

---

# Guiding Question

Before implementing any feature, ask:

> Does this make it easier for someone to pass their certification exam with confidence?

If the answer is no, reconsider the feature.
